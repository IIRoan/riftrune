# Product

## Register

product

## Users

Riftbound collectors and players using the app on mobile at shops, events, and trade tables, or on desktop while sorting binders, checking prices, and comparing printings. They need to find the right card fast, understand the exact version, and update ownership without ambiguity.

## Product Purpose

Riftbound is a card catalog and collection tracker. It should make card discovery, card details, variant comparison, price checks, and collection updates feel fast, trustworthy, and consistent across mobile and web.

## Brand Personality

Smooth, precise, and collectible. The interface should feel calm and utilitarian, with enough Riftbound artifact detail to avoid becoming generic dashboard UI.

## Anti-references

No gamer neon, no noisy card-game chrome, no generic SaaS dashboard palette, and no marketing-first screens that block users from the catalog. Motion should feel responsive, not theatrical.

## Design Principles

Start in the task: search, browse, inspect, and update collection state should be visible immediately.

Collector confidence: prices, printings, rarity, set, ownership, and auth state need clear hierarchy and stable formatting.

Version clarity: when a user acts on a specific printing, the app should preserve that exact printing instead of asking again.

Smooth restraint: use motion for state changes and orientation only, with reduced-motion support.

Personality through artifacts: let real card imagery, game symbols, and precise microcopy carry the brand instead of decorative effects.

## Accessibility & Inclusion

Target WCAG AA contrast. Support keyboard focus, readable text sizes, clear hit targets, reduced-motion preferences, and color-independent status labels for rarity, ownership, auth, and market states.
